Goal:
Create a Python-based web app (FastAPI + simple frontend) that lets a hardcoded user search and explore volunteer opportunities. The app will also support simple tools (quiz, hour tracker, reflections, share) and use the OpenAI API for AI-powered recommendation scoring.

⚙️ Backend Setup

Stack:

Python (FastAPI) for backend

Jinja2 templates or React for frontend UI

In-memory data storage (no database)

OpenAI Chat Completions API for recommendation scoring

Minimal authentication: single hardcoded username/password

Hardcoded credentials:

USERNAME = "student"
PASSWORD = "password123"

🧩 Backend Features

Authentication

Simple login page (username + password)

Store session in memory for the logged-in user

Search API

Endpoint: /search

Accepts a search query (e.g., “environment” or “tutoring”)

Returns a list of volunteering opportunities from a predefined dataset (in-memory)

Fields: title, location, hosted_by, requirements, description

AI Recommendation

Endpoint: /recommend

Takes user profile/interests

Calls OpenAI API to rank or summarize top opportunities

Returns scored opportunities

Tools API

/quiz → simple quiz to collect interests and personality type

/tracker → store volunteer hours in memory

/reflections → store user reflections (text entries)

/share → generate dummy shareable link

🧠 OpenAI API Integration

Endpoint /recommend should send user preferences to gpt-4o-mini (or similar model) to get ranked or summarized recommendations based on the volunteer data.

Example prompt:

"Given this user profile and the following volunteer opportunities, rank and explain the top 3 matches."
